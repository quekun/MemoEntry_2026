游戏基本功能全部实现，今天进行测试，优化不足之处，修补bug，反思游戏设计理念



1）添加时间停止：在出现关卡标题贴图的那几秒钟，玩家和敌人不能做出任何行为，做出了一个很酷的效果：先给第一人称视角特写，然后镜头拉远开始游戏，原理是：我使用时间暂停功能，在tank刚刚出生时，弹簧臂被“冻结”，还没来得及伸长，摄像头还位于actor原点，之后时间恢复流动，视角拉远了



2）优化按钮位置：当视口变大，变成整个屏幕时，按钮位置发生偏移，查阅资料发现是锚点位置未对齐，对齐后，即使分辨率变化，按钮位置不变



3）修复关卡逻辑bug：之前是通过禁用按钮实现，但是忽略了可以在游戏内的暂停界面通过打开下一个关卡，体会到了游戏开发的一句名言：“永远不要只在 UI 层面做限制”，最好通过把判断逻辑写在切换关卡的代码处，实现修复漏洞，毕竟是代码层面（我直接在蓝图里面修改了，hh）


4）为什么要到暂停界面做下一关的按钮？这个要求不是不合理吗


5）发现游戏结束后，设置的各种特效和音效就无法显示，因为直接切换到ui了，通过计时器延迟触发处理游戏结束函数实现优化


6）发现当敌人和玩家基本同时死亡，但玩家早一点点死亡时，还是算玩家胜利，回顾代码，发现因为代码里的 bool IsGameOver = false是一个局部变量，把它设置为成员变量，在TankHeroGameMode处理actor死亡的函数ActorDied中加上提前判断游戏是否结束，从而规避bug
